# 项目一 · 迭代三 冒烟测试用例

## 测试说明
- 测试类型：冒烟测试（Smoke Test）
- 测试目标：验证核心链路可用、页面稳定
- 执行环境：本地前后端（localhost）
- 执行耗时：约 5–10 分钟

## 冒烟测试用例表

| 用例ID | 用例名称 | 前置条件 | 操作步骤 | 预期结果 | 状态 |
|------|--------|--------|--------|--------|----|
| SMOKE-01 | 页面基础可用 | 前后端已启动 | 打开前端页面 | 页面正常渲染，无白屏、无报错 | ✅ |
| SMOKE-02 | 初始化加载碎片 | 当天已有碎片 | 刷新页面 | 自动加载今日碎片 | ✅ |
| SMOKE-03 | 提交单条碎片 | 页面正常 | 输入文本并提交 | 列表新增一条碎片 | ✅ |
| SMOKE-04 | 连续提交稳定性 | 无 | 连续提交 2–3 条 | 每次提交成功 | ✅ |
| SMOKE-05 | 刷新数据一致性 | 已有多条碎片 | 刷新页面 | 数据不丢失 | ✅ |
| SMOKE-06 | 未打卡轻提醒 | 打卡未完成 | 提交碎片 | 提醒但不阻断提交 | ✅ |
| SMOKE-07 | 打卡按钮无副作用 | 页面正常 | 点击"帮我打卡" | 不触发接口，仅提示 | ✅ |
| SMOKE-08 | Author 状态保持 | localStorage 有 author | 刷新页面 | author 自动恢复 | ✅ |

## 测试结论
- 迭代三冒烟测试全部通过
- 当前版本可作为稳定版本继续演进
